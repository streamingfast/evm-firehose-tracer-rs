# prost â†’ quick-protobuf migration

Replace `prost`/`prost-types`/`prost-build` with `quick-protobuf` + `pb-rs` for faster
protobuf serialization. Wire format is identical (proto3), so the test suite validates
correctness automatically.

## Background

`quick-protobuf` generated code is ~10Ă— smaller than prost and serializes faster because
it avoids reflection. Code is generated by `pb-rs` (the `quick-protobuf` codegen tool).

Reference: `streamingfast/substreams-ethereum` branch `feature/quick-protobuf` (PR #32).

---

## Key API differences

| prost (current) | quick-protobuf |
|---|---|
| `#[derive(prost::Message)]` | generated via `pb-rs` offline |
| `block.encode_to_vec()` | `Writer::new(&mut buf); block.write_message(&mut w)` |
| `Block::decode(&bytes)` | `Block::from_reader(&mut BytesReader::from_bytes(&b), &b)` |
| `Vec<u8>` for bytes fields | `Cow<'a, [u8]>` (owned = `Cow::Owned(vec)` / `.into()`) |
| `String` for string fields | `Cow<'a, str>` |
| `prost_types::Timestamp` | generated inline from `google/protobuf/timestamp.proto` |
| `#[derive(Default)]` on all structs | must implement `Default` manually or via `..Default::default()` pattern |

The Cow lifetime propagates to every struct that contains bytes or strings â€” essentially all
generated types carry `'_` or a named lifetime `'a`.

## Steps

### Step 1 â€” Acquire .proto files

No proto files currently live in the repo. Export them from the Buf registry:

```bash
# Requires buf CLI (https://buf.build/docs/installation)
buf export buf.build/streamingfast/firehose-ethereum --output firehose-tracer/proto
```

This pulls:
- `sf/ethereum/type/v2/type.proto`
- `sf/ethereum/substreams/v1/*.proto`
- `sf/ethereum/transform/v1/*.proto`
- Transitive dependencies (including `google/protobuf/timestamp.proto`)

Commit the resulting `firehose-tracer/proto/` directory.  Add a `gen.sh` at the repo root
documenting how to re-export if the proto schema ever changes.

### Step 2 â€” Add build dependencies

`firehose-tracer/Cargo.toml`:
```toml
[dependencies]
quick-protobuf = "0.8"

[build-dependencies]
pb-rs = "0.10"
```

Workspace `Cargo.toml`: add `quick-protobuf = "0.8"` to `[workspace.dependencies]`.

### Step 3 â€” Write `firehose-tracer/build.rs`

Follow the pattern from `substreams-ethereum/core/build.rs`:

```rust
fn main() {
    // 1. Collect all .proto files under firehose-tracer/proto/
    // 2. Strip options that pb-rs rejects (go_package, java_package, csharp_namespace)
    //    by pre-processing each file into a temp dir
    // 3. Run pb-rs Configuration to generate into src/pb/quick/
    // 4. Post-process the output to fix reserved keywords:
    //      "pub mod type {" -> "pub mod r#type {"
    //      "pub type: " -> "pub r#type: "  (struct fields named `type`)
}
```

Key pb-rs options to set:
- `gen_info = false` (suppresses file header comment)
- Output directory: `src/pb/quick/`

### Step 4 â€” Regenerate `src/pb/`

Run `cargo build -p firehose-tracer` once to trigger `build.rs` and generate
`src/pb/quick/` with the new module tree.

Review the generated types, paying attention to:
- `BigInt` â€” has `bytes: Cow<'a, [u8]>` (used heavily in balance/storage changes)
- `Block` â€” has `hash: Cow<'a, [u8]>`, `number: u64`, etc.
- `BlockHeader` â€” contains `timestamp: Option<Timestamp>` (now generated inline, not from prost-types)
- All enum variants â€” pb-rs generates enums differently from prost (no `i32` casts needed)

Update `src/pb/mod.rs` to expose the quick-protobuf module alongside or replacing the
prost one. The substreams-ethereum approach gated it behind a cargo feature; here we can
replace directly since this crate is the tracer itself (no external consumers of the prost
types).

### Step 5 â€” Update `src/pb/extensions.rs`

`BigInt` now carries a lifetime: `BigInt<'a>`. The `impl BigInt` blocks need updating:

```rust
// Before
impl BigInt {
    pub fn from_optional_u256(value: U256) -> Option<Self> { ... }
}

impl From<U256> for BigInt {
    fn from(value: U256) -> Self {
        BigInt { bytes: trimmed_bytes }
    }
}

// After
impl BigInt<'static> {
    pub fn from_optional_u256(value: U256) -> Option<Self> { ... }
}

impl From<U256> for BigInt<'static> {
    fn from(value: U256) -> Self {
        BigInt { bytes: Cow::Owned(trimmed_bytes) }
    }
}
```

Using `'static` (i.e. fully owned `Cow::Owned`) is the right choice here since we always
construct these from computed byte vecs, not borrowed slices.

### Step 6 â€” Update `src/mapper.rs`

`big_int_from_u256` returns `BigInt<'static>` â€” no change to callers, just the return type
annotation if explicit.

The `u256_trimmed_be_bytes` helper stays unchanged (returns `Vec<u8>`).

### Step 7 â€” Update `src/tracer.rs` (bulk of the work)

Every `Vec<u8>` field assignment in struct literals needs a `.into()` call to convert to
`Cow::Owned`. Examples:

```rust
// Before
Block {
    hash: block_data.hash.0.to_vec(),
    ...
}

Withdrawal {
    address: w.address.0.to_vec(),
    ...
}

// After
Block {
    hash: block_data.hash.0.to_vec().into(),
    ...
}

Withdrawal {
    address: w.address.0.to_vec().into(),
    ...
}
```

String fields follow the same pattern: `some_string.into()` or `Cow::Owned(string)`.

`..Default::default()` usage: pb-rs does **not** derive `Default` automatically. Either:
- Implement `Default` manually for `Block`, `TransactionTrace`, `Call`, etc. (verbose), or
- Expand the struct literals to explicitly set all fields (safer, removes the `Default` dep)

The substreams-ethereum branch took the explicit-fields approach. Recommend same here.

Enum field assignments change from `i32` casts to direct enum variants:

```rust
// Before (prost uses i32 for enums in struct fields)
call_type: CallType::Create as i32

// After (pb-rs uses the enum type directly)
call_type: CallType::CREATE   // pb-rs generates SCREAMING_SNAKE_CASE variants
```

Note: pb-rs generates enum variant names as SCREAMING_SNAKE_CASE vs prost's CamelCase.
Check the generated code for exact names.

### Step 8 â€” Update `src/printer.rs`

```rust
// Remove:
use prost::Message;

// Add:
use quick_protobuf::{MessageWrite, Writer};

// Encoding (replace lines ~68-71):
// Before:
let marshalled = block.encode_to_vec();

// After:
let mut marshalled = Vec::with_capacity(block.get_size());
let mut writer = Writer::new(&mut marshalled);
block.write_message(&mut writer).expect("protobuf encode failed");
```

`get_size()` is generated by pb-rs and returns the byte length of the encoded message â€”
use it to pre-allocate the buffer and avoid reallocs.

### Step 9 â€” Update `firehose-tracer-test/src/tracer_tester.rs`

```rust
// Remove:
use prost::Message;

// Add:
use quick_protobuf::{BytesReader, MessageRead};

// Decoding (replace lines ~360-366):
// Before:
let block = pbeth::Block::decode(&decoded[..]).expect("Failed to decode protobuf");

// After:
let mut reader = BytesReader::from_bytes(&decoded);
let block = pbeth::Block::from_reader(&mut reader, &decoded)
    .expect("Failed to decode protobuf");
```

Workspace `Cargo.toml` â€” remove:

`firehose-tracer/Cargo.toml` â€” remove `prost.workspace`, `prost-types.workspace`.

## Known risks / unknowns

| Risk | Mitigation |
|---|---|
| `google.protobuf.Timestamp` field access changes | Check generated `Timestamp` struct field names; `seconds`/`nanos` should be identical |
| pb-rs enum variant naming (SCREAMING_SNAKE_CASE) | Grep for all `as i32` casts and enum references in tracer.rs after generation |
| `Default` not derived | Expand all `..Default::default()` usages in tracer.rs into explicit field lists |
| Lifetime `'a` on nested types | Use `'static` throughout (Cow::Owned) since tracer always owns its data |
| `transform.v1` proto availability | Included in the buf export; verify it's generated correctly |
